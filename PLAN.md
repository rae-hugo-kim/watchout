# Galaxy Watch8 아빠용 학습 사이트 작업 계획

## 1) 목표
- 갤럭시 워치8 초보(부모님)가 **혼자 따라하면서 익힐 수 있는** 웹 가이드 제작
- 핵심 시나리오 우선:
  1. 기본 조작
  2. 전화 받기/끊기
  3. 내 핸드폰 찾기
  4. 건강 측정(심박/스트레스/체성분)

## 2) 사용자 기준
- 긴 글보다 큰 버튼 + 큰 글자 + 단계별 이미지 선호
- 실수 방지/복구 안내가 중요
- 한 페이지에서 한 가지 작업만 배우게 구성

## 3) 정보 구조(IA)
- `index.html` : 랜딩 + 전체 기능 카드 + 빠른 시작
- 섹션형 튜토리얼(한 페이지 내 앵커 이동)
  - 처음 세팅
  - 기본 조작
  - 전화 받기
  - 핸드폰 찾기
  - 건강 측정
  - 문제 해결 FAQ

## 4) 페이지 템플릿
각 기능은 동일한 템플릿 사용:
1. 이번 페이지 목표
2. 준비물
3. 따라하기 단계(1~5)
4. 체크리스트(완료 확인)
5. 문제 생기면(짧은 해결 팁)
6. 다음 단계 버튼

## 5) 디자인 원칙
- 최소 글자 20px, 버튼 높이 56px 이상
- 대비 강한 색상(가독성 우선)
- 전문 용어 최소화
- 모바일 우선 레이아웃

## 6) 구현 계획
### Phase A — MVP
- 정적 웹페이지 1개(`index.html`)로 빠르게 공개 가능한 형태
- CSS/JS 분리 (`styles.css`, `script.js`)
- 단계별 카드와 체크박스 인터랙션
- 이미지(일러스트 SVG) 포함

### Phase B — 콘텐츠 보강
- 실제 워치/폰 스크린샷으로 SVG 임시 이미지를 교체
- 아빠 테스트 후 문구 단순화
- 막히는 구간 FAQ 보강

## 7) 검증 체크리스트
- [ ] 모바일 화면에서 글씨/버튼 충분히 큰가?
- [ ] 첫 사용자 기준으로 3분 내 `전화 받기` 성공 가능한가?
- [ ] `내 핸드폰 찾기`를 혼자 실행 가능한가?
- [ ] 건강 측정 안내가 안전/주의사항 포함하는가?

## 8) 배포/운영
- 로컬 실행: `python3 -m http.server 4173`
- 저장소: `git@github.com:rae-hugo-kim/watchout.git`
- 이후 GitHub Pages로 배포 가능 (`main` 브랜치 기준)
